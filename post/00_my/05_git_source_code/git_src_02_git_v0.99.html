Readme

这个readme里介绍了很多的东西，主要介绍了The Object Database（Blob，Tree，Commit，Tag，把Trust也写了这一部分),然后就是暂存的概念The "index" aka "Current Directory Cache"，其中的工作流程就是working directory -> index -> object database或者反着来object database -> index-> working directory每个里面都说了具体使用的是哪条指令，例如 git-update-cache filename就是1) working directory -> index的具体操作。
如果需要检查数据，那么就用git-cat-file -t 及it-cat-file blob|tree|commit 来查看object。

Copyright

这里添加了一个copyright,可以看出来使用的GPL的开源许可协议，网上搜索了一下GPL，是GNU General Public License的缩写，是GNU通用公共授权非正式的中文翻译。看来这么多，感觉下面的对话说的应该就是实际应用的情况：

参考自 https://www.oschina.net/question/2349081_2304102
Chen-Zhiqiang 2019/03/06 12:29 使用GPL就要付出相应的代价，在遵循GPL协议的前提下，需要把所修改的源代码提供给那些安装了该软件的人，而且，GPL确实具有传播性，使用并修改了GPL许可的代码，那么该软件也应以GPL开源。然而，你的软件如果只是部署在自己的服务器上，并非是把软件代码本身给其他人使用，按GPL协议，那么其实就是需要把源代码按GPL许可开源提供给你自己，因为你才是安装了该软件的人。所以，在某种特定的场合下，倒也不必非要公布你的源代码。
昨夜今夕 2019/03/07 11:42如果不修改源码，只是使用了它呢，但是又不是在服务器内部，在浏览器端显示的效果 ，假设 jquery 是 GPL开源的，那么是不是所有使用了 jquery 的网站，都要把整个网站开源，因为它的网站也提供给外部人员使用了
Chen-Zhiqiang 2019/03/07 19:07 不管是否修改，用了GPL就得遵循它的规则。但问题是，你的软件只是将网站展示给别人浏览呢？还是你要出售你的软件？
GPL 只是规定用户在获取你的程序的时候必须可以获得源代码。如果你只是将网站展示给用户，而不是兜售你的使用了GPL代码的软件，那么实际上是不必开源的，因为按GPL来说它也只是开源给你自己，别人只是浏览该软件的展示效果，并不是要获取你的程序。
仅是个人理解。
